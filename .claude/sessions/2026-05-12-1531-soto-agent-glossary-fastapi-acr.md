## Key Learnings
- **`az acr build --file <path>` is cwd-relative, not context-relative.** Surprising — `docker build` is context-relative. Chased a .dockerignore red herring for ~10 min before catching this. Fix: `cd` to the intended context dir before invoking. Saved as memory `insight_az_acr_build_file_path.md`.
- **Glossary merge pattern.** When a term appears in both canonical tables AND Flagged Ambiguities, we surface BOTH. The first version returned only the canonical hit; pytest case `test_ambiguity_for_promotion` caught it. Good example of tests driving a real design refinement, not just regression coverage.
- **Page-selector hallucination is a real failure mode.** Stats-table category names (e.g., "Entities | 250") got read by the page-selector LLM as folder prefixes. Defensive `_resolve_page_path` fallback is now the safety net; hardened prompt is the prevention.
- **System-prompt vocabulary primer is doing real work.** First three smoke-test runs answered correctly without ever invoking `glossary_lookup` because the primer already covered "Rate of Sale", "Retailer Objective", etc. Good for demo quality, bad for verifying tool wiring — had to force an explicit instruction or pick a non-primed term to confirm the dispatch path.
- **Slim deps strategy.** Repo `pyproject.toml` carries torch/transformers/langchain for learning notebooks; the agent only needs 5 packages. A separate `soto_agent/requirements.txt` keeps the image ~150MB instead of ~5-8GB.
- **`async` doesn't matter inside the tool functions.** Local file I/O is microseconds. Async wins at the FastAPI route layer (uvicorn multiplexes users) and for fire-and-forget Path A PE publishes — not inside `glossary_lookup` or `wiki_read`.

## Decisions Made
- **Vault subset = build artifact, not committed.** Gitignored `soto_agent/data/vault/`. The source-of-truth is the user's local Obsidian vault; the recipe (`build_vault_subset.py`) is committed, the output is not. Same pattern as `node_modules/`.
- **Glossary = committed copy.** 137 lines, low volatility, makes the agent runnable + dockerizable without a sibling-repo path. `make refresh-glossary` syncs from BCC_NGST_CG when needed.
- **Denylist over allowlist for vault filtering** (user's call; explicit). Ship-by-default for BBC content; explicit `DENY_DIRS` + `DENY_FILE_GLOBS` for personal/learning. With `ALLOW_OVERRIDES` set for false positives like `AI in Beverage Alcohol Landscape.md`.
- **Filter the index, don't rebuild it.** Preserves the hand-curated one-liners that drive `wiki_search` relevance. Rebuilding from frontmatter would lose summary quality and tank tool effectiveness.
- **Makefile-orchestrated deploys.** User explicitly wanted automation so prep steps can't be forgotten. `make prep`/`make image`/`make deploy` chain everything.
- **`server.py` separate from `app.py`.** CLI entry stays in `app.py`; HTTP wrapper lives in `server.py`. Two surfaces, two files. `app.py` imports stay sync.
- **Slim Docker requirements** (separate `requirements.txt`, not derived from `pyproject.toml`). v2 in its own repo will get a clean dep tree by default; for v1 we work around the monorepo.

## Gotchas & Warnings
- `az acr build --file` cwd resolution (see Key Learnings). Captured in memory.
- `BaseModel` had to be imported in `app.py` once `_serialize_tool_result` was generalized — pre-existing `WikiPage` check didn't need it. Easy to miss in a refactor; the resulting `NameError` was caught by `try/except` and returned to the model as a bogus tool result, masking the real failure mode (no tool call appeared to fire).
- `logger.log(obj)` is a TypeError trap — `Logger.log` requires `(level, msg)`. Use `logger.info`/`logger.exception` instead. User hit this in `server.py`.
- FastAPI route signature: param annotated as Pydantic BaseModel binds to JSON body; plain `str`/`int` binds to URL query string. Easy to wire a route that "works" but reads from the wrong place.
- `.dockerignore` `* + !soto_agent/` interacted oddly with `az acr build` here; switched to explicit excludes. Worth re-investigating later — may have been the cwd-relative bug masquerading.
- "Promotion" exists in BOTH canonical tables and Flagged Ambiguities. Any term-lookup design that returns only the first hit will miss the ambiguity guidance — surfaced by pytest, not by manual smoke testing.
